# e058 渦ダイポールepisodeの打ち切り対応生存時間分析（H021キャンペーン反復7・role V）

外部監査2件（GPT・別Claude、2026-07-16）が e055/e056 の測定手法に指摘した統計的弱点——「独立 seed 集合の
最大値比較」「観測窓終了時点で束縛中のペアを死亡として扱っている（打ち切りの誤処理）」——を、Observer v2
（`vortex_tracking_v2` のグローバル割当トラッカー＋`dipole_episodes` のフレーム毎 episode 検出＋
`survival_analysis` の Kaplan-Meier）で正しく再測定する。

## マージ前の追加外部レビュー（Codex・CodeRabbit、2026-07-16）で発見・修正した4件のバグ

本PR（このexperiment.yamlをまだマージしていない時点）に、GitHub上でCodex・CodeRabbitという2つの独立した
外部AIレビューが自動で届き、Observer v2の実装そのものに4件の具体的なバグを指摘した。両者が独立に同じ
問題に収束したため、下記の測定結果に反映する形でマージ前に修正した（**マージ済みの数値を後から訂正した
のではなく、これが本実験の最初で唯一の公開測定値**）:

1. **`dipole_episodes.py`：フレーム毎に「一番近い1組」しか記録していなかった**。同一フレームに複数の
   独立した±束縛ペアが同時に存在し得る（例えば箱の中の別々の場所で2組が同時に束縛している）のに、
   単一のグローバル最近接ペアしか保持しない設計だったため、他のペアを黙って見落としていた。修正：
   各フレームで「相互最近傍」（正トラックの最近傍が負トラックX、かつXの最近傍がその正トラック）となる
   ペアを全て検出し、複数の episode を同時並行で追跡できるようにした。
2. **`survival_analysis.py`：RMST計算が最後の観測時刻からtauまでのKM平坦区間を積分し忘れていた**。
   tauが最後の観測時刻より先の場合、その区間の生存確率（平坦線）を足し漏らし、RMSTを過小評価していた。
   修正：`surv[-1] * (tau - times[-1])` を明示的に加算。
3. **本ファイルの元実装：frame 0で既に束縛済みのepisodeを「観測開始時点が既知の完全な生涯」として
   扱っていた**。トラッキングはクエンチ＋hold後にしか始まらないため、frame 0時点で既に束縛していた
   ペアはトラッキング開始前に形成された可能性があり、真の開始時刻は不明（左側打ち切り）。それを既知の
   開始時刻として扱うと、観測された残り寿命だけを生涯全体であるかのように数え、生存時間分布を下方に
   バイアスする。修正：`start_frame == 0` のepisodeは除外し、除外件数を明示的に報告する。
4. **`vortex_tracking_v2.py`：Hungarian割当をmax_step超過ペアのマスクなしで解いていた**。大域最小コスト
   解が「コストは低いがmax_step超過で後で棄却されるペア」を選び、本来ならmax_step内で成立していた別の
   ペアの枠を奪ってしまうケースがあり得た（例：コスト行列`[[10,9],[9,0]]`でmax_step²=9のとき、無マスクの
   Hungarianは合計10=`(10,0)`を選ぶが、両方ともmax_step内に収まる合計18=`(9,9)`の方が2トラックとも
   有効に継続できる）。修正：解く前にmax_step超過ペアを大きな値でマスクし、実行可能な解を優先させる。

いずれも本PRでの初回実装（PR-A・PR-B内）のバグであり、既存のe052〜e057・e055/e056の元データには一切
関係しない。修正後のコードで `pytest tests/test_vortex_tracking_v2.py tests/test_dipole_episodes.py
tests/test_survival_analysis.py tests/test_e058_dipole_survival_km.py` を実行し、新規回帰テスト2件
（同時多重ダイポール検出・RMSTのtau延長）を含めて全19件通過を確認した。

## 測定結果（full・L=96・τ_Q=200・観測窓1600 step=201 snapshot・20 seed 独立新規範囲）
- **182 episode**（20 seed 合計、frame 0で既に束縛済みだった209件は左側打ち切りとして除外）。
- **KM 中央値生存時間 = 62 frame**（物理時間 24.8）。
- **制限付き平均生存時間 RMST = 57.84 frame**（tau=170 frame まで）。
- **打ち切り割合 = 14.8%**（182 中 27）——観測窓は大半の episode の生涯を捉えるが、7件に1件強はまだ
  束縛中に窓が終わっている。
- 分布の形：中央値62 frame、最長170 frame（打ち切り）という幅を持つ。

## 正直な評価
1. **打ち切りは無視できるほど小さくはない**：14.8% という打ち切り割合は、e055/e056 が懸念した
   「観測窓が短すぎて死亡と生存を区別できない」という問題が、この観測窓長（baseline の1600 step）
   でも完全には解消されていないことを示す。
2. **「無限持続の証拠なし」という e056 の定性的結論を、本実験1本だけで確定させるのは正確でない**：
   打ち切り率14.8%は、真に長寿命な個体が紛れ込んでいる可能性を排除できるほど低くない。baseline窓
   （1x）だけを測った本実験は、e055/e056が本来問うていた「窓を延ばしても最長寿命は伸びないか」という
   飽和問題そのものには直接答えていない——e055/e056スタイルの窓長スイープ（3倍・6倍・9倍）をObserver v2
   で正しくやり直すことが必要（次の反復候補）。
3. **【重要】v1(e055/e056)との数値差は矛盾ではない——測定対象が違う**：
   - v1 の `mutual_neighbor_durations` は「2つのトラックが共存する全期間にわたり相互最近傍であり続けたか」
     を判定する。この設計は、そもそも**ある程度長く共存したトラックペア**にしか適用されない（片方の
     トラックがすぐ消えれば重複期間自体が短く、"duration" として記録されにくい）。
   - v2 の `dipole_episodes` は**フレーム毎**に相互最近傍を判定し、一瞬だけ最近傍になった後すぐ離れる
     ような短命な束縛イベントも**全て**episodeとして数える（バグ修正後は同時に複数存在する束縛ペアも
     全て数える）。
   - 結果、v2 の母集団は v1 とは異なる、より完全な量を測っている。「短くなった」「長くなった」の
     単純比較ではなく、測定対象自体が異なることを踏まえて解釈する。

## Discipline
- **role V（単独、secondary N は付けない）**：打ち切り率14.8%を踏まえると、「無限持続の証拠なし」という
  negative_constraint 主張をこの1実験だけで確定させるのは過大評価になる。新しい発見というより、
  e055/e056 の問いを正しい手法で再測定する検証実験として role V に留める。
- **e055/e056 の元データ・artifact は一切変更していない**（no_touch）。本実験は独立した新規測定
  （新規 seed 範囲2001-2020、e052〜e057 のいずれとも重複なし）。
- **物理は完全凍結・再利用**：e052 と同一の damped GPE KZ クエンチ。新しい propagator は追加していない。
- **8th 監査**：`target_encoded=false`。

## Observer v2 の設計修正（開発中の発見、本PR内で2段階に分けて実施）
第1段階（自分自身の実データ検証で発見）：`dipole_episodes.detect_episodes` の初期実装は、フレーム毎の
運動学的チェック（COM速度と分離ベクトルの直交性・分離変化率）を**episode継続のハード条件**にしていたが、
これは単一ステップの位置決定ノイズに敏感で、本物の並進ダイポールでも頻繁にepisodeを分断してしまうバグを
合成テストでなく実データで発見した。修正：episode の継続条件は「パートナー同一性＋分離距離の上限」のみとし、
運動学的な質（直進性・分離安定性）はepisode全体の要約統計（`straightness`・`mean_sep_rate`・
`kinematics_ok`）として報告する設計に変更した。

第2段階（外部AIレビューで発見、上記4件）：単一最近接ペアのみの検出・RMST積分漏れ・frame 0左側打ち切り
未対応・Hungarian割当のマスク漏れを修正。
