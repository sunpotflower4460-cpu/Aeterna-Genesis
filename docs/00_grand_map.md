# 00 — 全体地図（背骨と、入力を剥がす順番）

一行で：**本物の場から、最小条件で、構造が勝手に出るのを、掟つきで一個ずつ
積む。** それが Aeterna-Genesis。

## 背骨

宇宙が始まるとき、おそらく最小条件しかない。その条件から、意図せずに構造が
"勝手に" 生まれた。それを、この場で、本物としてやる。観測される要素（幾何・
物質・渦・トーラス・重力…）が、作為なく、最小条件から創発するのを、一個ずつ
確かめて積む。

- **(A) 忠実な創発**：実在の場の方程式を入れ、最小条件から現象が勝手に出る。
  **今すぐ厳密にできる。** ここを厳密に積む。
- **(B) 法則を根から導く**：場の方程式さえ 0≠無 から出す。最深の目標。**まだ。**

各モジュールは「今これは (A) か (B) か、どの入力をまだ手で入れているか」を
必ず明記する（LAW.md §1 の `A_OR_B`）。

## 入力を剥がす順番（各段は前段の core を再利用）

```
e001  GPE 単一渦の歳差        [GREEN]  入力: 場の方程式 + 渦1個
  └ 場だけから「動き」が出る。歳差・半径一定・循環量子化を measured で確認。

e002  GPE 二渦の相互作用       [GREEN]  入力: 場の方程式 + 渦2個
  └ 同符号→回り合う(共回転322.9°)／逆符号→対で並進(中心移動21)。循環保存を measured で確認。
     クリーン窓は間隔で自己検出（劣化前で打ち切り）。連続性追尾で恒等保持。

e003  GPE 3D 渦リング(トーラス) [計画]   入力: 3D 場の方程式 + リング1本
  └ リングが伝播・絡む(リンク)か。閉じた渦線＝トーラス。

e004  白→形(場+不安定+ノイズ)   [計画]   入力: 場 + 不安定 + 一様ノイズ
  └ 対称が自分で破れ欠陥が生まれる(TDGL)。e001 より入力が少ない＝より深い。

e005  因果順序→次元           [計画]   入力: 因果集合(座標なし)
  └ 次元(Myrheim-Meyer)が出る。幾何の創発。

e006  物質⇄重力(場ベース)      [計画]   入力: 場 + 幾何の相互作用
  └ 場の構造が自分で重力を生み、幾何が場に返る共進化。

e0xx  重力の創発 ((B)へ)       [frontier] 誘導重力をエンタングルメントから。
```

各段で「まだ手で入れている入力」を明記し、次段でそれを剥がす。これは
"届く範囲のリスト" ではなく、白から全部が立ち上がるのを実際に見にいく順番。

## 共有の道具（core/）

| ファイル | 役割 |
|---|---|
| `core/fft.py` | 2D FFT・波数格子（split-step の運動項） |
| `core/field.py` | 複素場の生成、TF初期化、トラップ V、規格化、split-step 伝播 |
| `core/vortex.py` | 位相巻き数による渦検出、循環の量子化チェック |
| `core/measure.py` | 保存量（ノルム・エネルギー）、累積回転、（後で）次元・相関 |

実験は道具を組み立てるだけ。実験ごとに FFT や物差しを作り直さない。
