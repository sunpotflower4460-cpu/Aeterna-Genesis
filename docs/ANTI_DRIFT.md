# ANTI_DRIFT — 答えを置くな。答えが「生まれる」ようにせよ

> **これは何か**：このプロジェクトで働く AI（Claude / Claude Code / Codex / Sonnet / GPT / その他）が
> Level を扱うとき、**答え（欲しい性質そのもの）を初期条件に置いてしまう “drift”** を防ぐガード。
> 両リポジトリ（`Aeterna-Genesis` / `Genesis-Room-`）共通で最初に読む。詳細は
> [`EMERGENCE_LEVELS.md`](EMERGENCE_LEVELS.md) / [`honest_floors.md`](honest_floors.md) /
> [`traps_museum.md`](traps_museum.md) / [`PIECES.md`](PIECES.md)。

## 合言葉（毎回これを通す）

> **それは生まれたのか、置いたのか?（Did it GROW, or did I PUT it there?）**
> **その部品、どこから持ってきたの?（Where did that come from?）**

drift は静かに起きる。「もう一段深い Level を出したい」→ つい欲しい性質を IC に足す → measured に見えるが
**それは"置いた"だけ**。気づかないと堂々巡りになる。だからこの一枚を毎回通す。

---

## 原則1：答えを置くな。答えが「生まれる」ようにせよ

**喩え（種と果実）**：リンゴの木に実をならせたい。実が**種から育って**なるなら本物。だが、買ってきたリンゴを
枝に**くくりつけた**ら、誰かに「**そのリンゴ、どこから持ってきたの?**」と聞かれる——木が育てたのではなく、
置いただけだ。**「必要だから置く」は、この“買ってきたリンゴをくくりつける”のと同じ。**

- 対応：**種 ＝ 初期条件（白）** ／ **実が育つ ＝ 深い性質（遺伝・循環・次元…）が創発する** ／
  **買ってきた実をくくりつける ＝ その性質を出すために場や項を足す**（例：遺伝を出すためにタグ場 `T` を足す）。
- 宇宙も同じ。「これがないから足す」ではない。**白（すべての色）から生まれたものが、宇宙・地球・生命の器に
  なった。** だから「足す」という発想自体が根本的にズレている。
- **欲しいもの（遺伝・循環・次元…）があっても、あくまで初期条件（種）からの“創発”で生まれさせる。**
  答え（それを起こす部品）を直接置かない。

## 原則2：初期条件を「問い直す」のはいい。だがそこにも罠がある

初期条件を変えるのは、「**どうしたら今の宇宙になるか**」がそのやり方でできないと思った時の、**別条件の仮説**
として。ただし——

- ❌ **問い直しの罠**：「遺伝が出ない → 遺伝が出やすい条件を仕込む」＝ 形を変えただけの「答えを置く」。
- ⭕ **問い直しの正しい形**：「重力が出ない → **なぜ出ないのか?**」「次元が出ない → **なぜ出ないのか?**」。
  **本来あるはずのものが出ないなら、「白」の考え方が少し違うのか?** と、**白の根源的な捉え方**へ矛先を向ける。
- ＝ 問い直す対象は「**答えを仕込む条件**」ではなく「**白そのものの捉え方**」。

## 原則3：「足りない/多い」を問うのはいい。答えを置くのはダメ

「この初期条件では特定のものが生まれ得ない」と分かった時、「**何が足りないか / 多いのか**」を問うのは OK。
ただしその答えが「**具体的な部品（遺伝場など）を置く**」になったら罠。
**「足りない/多い」は白の在り方への問い**であって、**答えの配置ではない。**

## 原則4：「どの部品を選ぶか」すら、まだ足す発想に近い（最深 frontier）

- (A) **白にもともと内在する自由度の顕在化**（複素場の位相 → 渦、U,V の濃度 → スポット）と、
  (B) **白になかった構成要素を増やす**（U,V → U,V,T）——**この区別すら、"どの部品を置くか"を人が決めている
  点で足す発想に近い。**
- 本来は「**一つの根源的な白から、宇宙も生命の器も生まれてくるか**」を問う。部品を選んだ時点で白から離れている。
- ⚠️ ただしこれはプロジェクト最深の **frontier**（「一つの根源から全部」は未証明の夢）。だから**現実の鉄則は
  「答えを置かない・出るように仕込まない」を必ず守り**、その上で「生まれ得る初期条件（白の捉え方）」を問う。

## 創発 vs drift（対照表・修正版）

| | ⭕ 創発（生まれた・measured） | ❌ drift（置いた・target_encoded / 別の白を選び直した） |
|---|---|---|
| 由来 | 白（種）に内在する自由度が **t=0 から**育つ | 欲しい性質を **場/項として足す・IC に埋める** |
| IC（白） | 一様・ノイズ・対称・（白に内在する）位相/濃度のみ | 目標パターン・巻き数・分裂後の配置・**遺伝場 `T` を足す** |
| 例（L2） | 一様＋ノイズ → 位相巻き渦が育つ（位相は複素場に内在） | `vortex_charges` で巻きを種に置く |
| 例（L3） | REST＋ノイズ → 循環ロールが自発 | 初期速度場・回転方向を与える |
| **例（L7）** | U,V ノイズ種 → スポットが**自己複製・分裂**（＝白に内在） | **遺伝が欲しいから遺伝場 `T` を足す ＝ 木に買ってきたリンゴをくくりつけた**（木が育てた実ではない） |
| 判定 | 第8監査 clean・tier=measured | 「その部品どこから来たの?」＝ 置いた／別の白を選んだ |

> ⚠️ 旧版の誤り：`founder タグ` を「Gray-Scott の IC」欄に入れていた。**タグ場 `T` を足すこと自体が
> "答え（遺伝）を置く"**（原則1）。founder に種付けしようが、`T` という自由度を人が足した時点で、
> それは Gray-Scott(U,V) の白からの創発ではない。→ 対照表・L7 ラベルを修正した（下記）。

---

## L7 のラベル（修正・正直な後退）

- **Gray-Scott(U,V) の天井 ＝ L7-partial（自己複製・分裂のみ）。** これがこの白の**正直な到達点**。
  スポットは白（U,V の濃度）に内在するものから生まれる＝創発。だが**遺伝（heredity）は生まれない。**
- **タグ場 `T` を足した系（U,V,T）は「答え（遺伝）を置いた／別の白を選び直した」**もの。
  daughters が親のタグを継承する“動き”は測れるが、**その heritable な自由度そのものを人が足している**ので、
  **Gray-Scott(U,V) の白から遺伝が"生まれた"のではない。** よって：
  - **「Gray-Scott が初期条件だけで FULL L7 に到達」は取り下げ**（retracted）。
  - コード/台帳は「Gray-Scott(U,V) = **L7-partial**」を天井とし、T 系は「**遺伝場を置いた別の genesis
    条件**（創発でなく配置）」と正直にラベル（full-L7-emergence は主張しない）。
- **「白から遺伝が生まれ得るか」は保留＝ frontier**（原則4）。出ないなら「この白の天井」として正直に記録する。

## 「生まれ得る条件」を問い直す正しい手順

特定のもの（遺伝・重力・次元…）が出ない時：

1. **(a) なぜ出ないのかを問う**——測定で機構を調べる（成長率・S(k)・循環・スポット数・欠陥…）。
2. **(b) 白の捉え方を問い直す**——「**本来あるはずのものが出ないなら、白の根源的な在り方（どんな自由度が
   "もともと"あるべきか）が違うのか?**」。矛先は**白そのもの**であって、答えを仕込む条件ではない。
3. **(c) 答え（部品）を置いて回避しない**——出ないなら「**この白の天井**」として正直に記録（負の結果・床は
   積み上げる）。「出やすい条件を仕込む」に逃げない。

## 「この Level を主張してよいか」YES チェックリスト

各項目 **YES** でなければ主張しない（NO があれば **STOP** → 白を種/ノイズ/対称へ戻し 0 から回し直す）。

- [ ] **t=0 から** run したか？（途中状態から始めていないか）
- [ ] 欲しい性質を**場/項/配置として足していない**か？（原則1：遺伝場 `T`・目標パターン・巻き数・娘への代入）
- [ ] その構造は**白に内在するものから育った**か？（"その部品どこから来たの?" に「白から」と答えられるか）
- [ ] その構造は**測定で現れた**か？（画像の印象でなく数）
- [ ] Level は **measures**（`assess_level` / `higher_levels`）で判定したか？（自作の成功ゲートで底上げしていないか）
- [ ] 揃わないなら **partial** と呼んだか？／出ないなら**この白の天井**として正直に記録したか？
- [ ] **同じ数学 ≠ 同じもの**：tier（`measured` / `interpretive` / `analogy` / `frontier`）を付けたか？
      （渦線≒宇宙ひも・spots≠life・もつれ→重力＝線形化 は **analogy**。生命語・断定語を使わない）

> 迷ったときの合言葉：**「それは生まれたのか、置いたのか?／その部品、どこから持ってきたの?」**
